1. 프로젝트 개요

본 프로젝트는 사용자가 선호 지역을 등록해 두고, 해당 지역의 현재 날씨·단기 예보·기상특보·의상 추천·날씨 랭킹을 한 화면에서 확인할 수 있는 웹 기반 대시보드를 구현하는 것을 목표로 하였다.

기상청 OpenAPI(초단기실황·동네예보·기상특보)를 통해 실시간·예보 데이터를 수집하고, Google Charts로 24시간 기온/강수확률/습도 변화를 시각화하여 데이터에 기반한 의사결정(외출 시간, 옷차림 선택 등)을 지원한다.

또한 선호 지역을 다단계(시/도 → 시/군/구 → 동/읍/면)로 선택·저장·삭제할 수 있는 UI를 제공하고, 다크모드/라이트모드 전환과 반응형 레이아웃을 통해 다양한 사용 환경(PC·태블릿·모바일)에서도 일관된 사용자 경험을 제공하도록 설계하였다.

화면설계 3 (날씨 랭킹 화면) 90 지역별 현재 기온을 순위 형태로 보여주는 카드 리스트 UI를 설계하였다. 구현에서는 랭킹 패널 틀과 카드 디자인을 완성하고, 서버 사이드에서 랭킹 데이터를 가져와 렌더링하는 구조까지 구축하였다.
화면설계 4 (반응형·다크모드 레이아웃) 95 1080px, 640px 이하에서 레이아웃이 세로 스택 구조로 변경되고, 상단 네비게이션의 텍스트가 숨겨지는 모바일 UI를 설계하였다. CSS 변수와 data-theme 속성을 활용해 라이트/다크 테마를 모두 구현하였다.
추가화면 1 (외부 링크 드롭다운) 100 상단 네비게이션 우측에 기상청/에어코리아/지도 등 외부 서비스를 모아 둔 드롭다운 메뉴 화면을 설계하고, 호버·열림/닫힘 인터랙션까지 구현하였다.
추가화면 2 (기상특보 상세 보기) 90 지역별 기상특보 리스트와 "더보기/접기" 토글이 포함된 상세 보기 화면을 설계하였다. 실제 구현에서는 최근 특보를 기본으로 보여주고, 버튼 클릭 시 나머지 특보를 슬라이드 형태로 펼쳐 볼 수 있도록 구성하였다.
기능목록
기능목록 1 (로그인·세션 관리) 100 세션 기반 사용자 인증을 구현하여 로그인하지 않은 사용자는 자동으로 로그인 페이지로 리다이렉트되도록 처리하였다. 로그인에 성공한 사용자만 대시보드에 접근할 수 있도록 함으로써 기본적인 보안 요구사항을 충족하였다.
기능목록 2 (선호 지역 저장·관리) 100 사용자별 선호 지역을 DB 테이블(user_regions)에 저장하고, 목록 조회·선택(보기)·삭제 기능을 제공하였다. 3단계 드롭다운(시/도 → 시/군/구 → 동/읍/면)을 통해 지역을 선택하면, PHP 스크립트를 통해 좌표(nx, ny) 및 기상특보용 코드(stnId)가 함께 저장된다.
기능목록 3 (실시간·예보 데이터 연동) 100 기상청 초단기실황(getUltraSrtNcst)과 동네예보(getVilageFcst) OpenAPI를 호출하여, 현재 기온·습도·풍속·강수형태와 향후 24시간 예보 데이터를 수집하였다. API 결과코드·응답 실패에 대한 예외 처리를 추가하여 오류 시 사용자에게 안내 문구를 출력하도록 구현하였다.
기능목록 4 (날씨 차트 시각화) 100 Google Charts의 ComboChart를 사용하여 시간대별 기온(선), 강수확률(막대), 습도(점선)을 동시에 시각화하였다. 다크모드 여부에 따라 배경색·선 색상을 동적으로 변경하여 가독성을 높였으며, 데이터 누락 시 이전 값을 보정해 그래프의 연속성을 유지하였다.
기능목록 5 (기상특보 조회) 95 시·군·구별 기상특보 API(getWthrWrnMsg)를 호출하여 최근 5일간 특보를 조회하고, 대표 특보 1건과 나머지 특보를 구분해 보여주는 UI를 구현하였다. "더보기/접기" 버튼을 통해 추가 특보를 동적으로 열고 닫을 수 있다.
기능목록 6 (오늘의 옷차림 추천) 100 현재 기온을 인자로 받아 계절·온도 구간별 의상 추천 문구를 반환하는 getClothingRecommendation() 함수를 별도 모듈로 분리하여 활용하였다. 추천 문구는 인용구 카드 형태로 표시되며, 온도 변화에 따라 실시간으로 내용이 달라진다.
기능목록 7 (지역별 날씨 랭킹) 90 별도의 PHP 스크립트에서 다수 지역의 현재 기온을 수집·정렬하여 상위/하위 지역을 산출하고, 프론트엔드에서는 Fetch API로 해당 HTML 조각을 불러와 "날씨 랭킹" 페이지에 동적으로 삽입하는 구조를 구현하였다.
추가기능 1 (다크모드·테마 유지) 100 HTML 루트 요소의 data-theme 속성과 CSS 변수를 활용해 라이트/다크 테마를 구현하였고, 사용자가 선택한 테마를 localStorage에 저장하여 새로고침·재로그인 이후에도 동일한 환경이 유지되도록 하였다.
추가기능 2 (반응형 UI 및 접근성 개선) 90 미디어 쿼리를 활용해 화면 너비에 따라 레이아웃과 텍스트 노출 방식을 변경하였다. 버튼에 title/aria-label을 부여하고, 포커스 스타일을 정의하여 키보드 사용성과 접근성을 일정 수준 확보하였다.
검증시나리오
검증시나리오 1 (로그인 및 접근 제어) 100 로그인하지 않은 사용자가 대시보드 URL에 직접 접근할 경우, 세션 검사 후 로그인 페이지로 리다이렉트되는지 테스트하였다. 정상 로그인 후에는 세션에 사용자 ID가 저장되고, 새로고침해도 세션이 유지되는 것을 확인하였다.
검증시나리오 2 (선호 지역 추가·삭제·선택) 100 서로 다른 시/도·시/군/구·동을 선택해 선호 지역을 여러 개 추가한 뒤, "보기" 버튼 클릭 시 해당 지역의 날씨 정보와 차트가 올바르게 변경되는지 검증하였다. 또한 삭제 버튼 선택 후 해당 지역이 DB 및 화면에서 모두 제거되는 것을 확인하였다.
검증시나리오 3 (기상 데이터 및 차트 표시) 95 다양한 시간대에 접속하여 현재 기온·습도·풍속이 정상적으로 표시되는지, 24시간 차트에 최소 1일 분량의 데이터가 누락 없이 표현되는지 확인하였다. API 호출 실패 상황을 가정하여 오류 메시지가 사용자에게 안내되는지도 함께 검증하였다.
검증시나리오 4 (기상특보·옷차림·랭킹) 90 특보가 있는 지역과 없는 지역을 각각 선택하여 특보 리스트·없음 안내 문구가 올바르게 출력되는지 확인하였다. 온도가 크게 다른 날(한파/폭염)에 접속하여 옷차림 추천 문구가 상황에 맞게 변경되는지 점검하였고, 랭킹 페이지에서 Fetch 오류 발생 시에도 에러 메시지가 노출되는지 테스트하였다.

3. 추가사항 (우월성을 증명할 수 있는 내용)

(1) 실제 공공데이터를 활용한 서비스
단순 더미 데이터가 아닌 기상청 OpenAPI를 실시간으로 호출하여 실제 환경과 동일한 데이터를 기반으로 동작한다는 점에서, 서비스 완성도와 실용성이 높다.

(2) 테마·반응형 UI 구현
CSS 변수·다크모드·반응형 레이아웃을 적극적으로 활용하여, 최신 웹 서비스에서 요구되는 사용자 경험(UX)을 충실히 반영하였다.

(3) 모듈화·확장성을 고려한 설계
날씨 데이터 수집, 기상특보, 의상 추천 등의 기능을 개별 함수·파일로 분리하여 유지보수성과 확장성을 높였다. 추후 다른 API(Heat Index, 미세먼지, 자외선 등)를 추가하기에도 용이하다.

4. 추가사항 (기술적 구현상의 특징)

(1) 동적 페이지 전환 및 부분 갱신
전체 페이지 새로고침 대신 JavaScript로 페이지 섹션을 전환가능하게 구현하였고, 랭킹 영역은 Fetch API를 통해 필요한 부분만 갱신하도록 구현하여 빠른 페이지 로딩을 구현하였다.

(2) 비밀번호 암호화 처리
회원가입 및 로그인 기능에서 사용자 비밀번호는 평문이 아닌 해시값(SHA-256)으로 암호화 처리되어 저장함
비밀번호 검증 시에도 해시값을 비교하여 인증을 수행한다. 따라서 해킹 시도시 실제 비밀번호 유출 위험을 원천 차단할 수 있다.

(3) 로그아웃 후 캐시 제어 및 세션 보안 강화
로그아웃 후 브라우저의 '뒤로 가기' 버튼을 클릭했을 때 이전 개인정보 화면(대시보드)이 노출되는 보안 취약점을 방지하기 위해, PHP 헤더(Cache-Control: no-store 등)를 설정하여 브라우저 캐싱을 강제로 차단하였다. 이를 통해 페이지 진입 시마다 서버 측 세션 유효성 검사를 필수적으로 수행하게 하여, 세션이 만료된 상태에서의 비정상적인 접근을 원천적으로 차단하였다.
