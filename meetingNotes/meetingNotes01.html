회의내용

■ 서비스 개요 논의

사용자가 로그인 후 자신이 설정한 지역의 기상 정보를 확인할 수 있는 웹 서비스를 개발하기로 결정하였습니다.

■ 주요 기능 정의

  1. 지역 기반 예보 제공
    • 사용자별 기본 지역 설정 기능
    • 복수 관심 지역 저장 기능(DB 연동)
  2. 기상청 단기예보 API 연동
    • 초단기 예보 데이터 수집
    • 단기 예보 데이터 수집
  3. 시각화 대시보드 구성
    • 기온 / 강수확률 / 습도 / 풍속 시각화
    • 구글 차트 기반 그래프 출력

■ 차별화 기능 논의

  1. 날씨 기반 옷차림 추천
    • 기온 기반 의상 추천 문구 제공
    • "오늘은 겉옷을 챙기시는 게 좋겠습니다."
  2. 지역별 날씨 랭킹 제공
    • 최고 기온/최저 기온 순위 제공
    • 관심 지역 간 비교 가능
  3. 위험 알림 기능
    • 미세먼지·폭염·한파 등 위험 요인 알림
    • "오늘 미세먼지 농도가 높습니다. 마스크 착용을 권장합니다."

■ 기술 스택 논의

프론트엔드, 백엔드, DB 구조 담당을 정하고 향후 세부적으로 수정하기로 했습니다.

■ 향후 일정

  1. API 테스트 및 데이터 구조 분석
  2. 대시보드 UI 제작
  3. DB 구성
