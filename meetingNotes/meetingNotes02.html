제안서 제목 데이터에 기반한 대시보드 작성
제안서 내용

1. 프로젝트 개요

본 프로젝트는 사용자 맞춤형 날씨 정보를 제공하는 웹 애플리케이션을 개발하여 데이터 기반의 실시간 대시보드를 구현하는 것을 목표로 한다.
기존의 단순 예보 제공 웹 서비스와 달리, 사용자가 설정한 지역을 기반으로 세부적이고 개인화된 날씨 데이터 및 위험 알림 기능을 제공한다.
또한, 데이터 시각화를 통해 예측 정보의 가독성을 높이고, 실제 사용자 생활에 도움이 되는 ‘날씨별 옷차림 추천’ 등 차별화된 부가 기능을 구현한다.

2. 데이터 설명

• 기상청 단기·초단기 예보 API 활용 (기온, 습도, 강수확률, 풍속 등 실시간 데이터)
• 지역별 날씨 지표 수집 후 자체 랭킹 로직에 활용
• 위험도 알림을 위한 미세먼지/강풍 등 외부 환경 데이터 추가 연동 가능
• 데이터는 시각화를 위해 Google Chart와 연동하여 대시보드 형태로 출력한다.

3. 화면설계 ※ 화면별 평가

3.1 로그인/회원가입 화면 – 최소 정보 입력 후 개인화 기능 제공

3.2 메인 대시보드 화면 – 지역 설정, 주요 기상정보 표시, Google Chart 시각화

3.3 추가 기능 섹션 – 옷차림 추천, 지역 랭킹, 위험 알림 리스트

4. 기능목록 ※ 기능항목별 평가 시나리오

4.1 회원가입 및 로그인
- 사용자가 계정을 생성하면 개인 지역 정보 저장 가능
- 로그인 후 개인화 된 대시보드 접근

4.2 개인 맞춤 날씨 정보 제공
- 사용자가 설정한 지역의 실시간 기상데이터를 API 통해 가져옴
- 기온, 강수확률, 습도, 풍속을 Google Chart로 시각화

4.3 날씨별 옷차림 추천 기능
- 설정 지역의 체감온도/기온에 따라 간단한 문구 제공 (“겉옷 챙기세요” 등)

4.4 지역별 날씨 랭킹
- 전국(또는 사용자가 선택한 범위)의 최고·최저 기온 지역 비교 및 순위화

4.5 위험 알림 기능
- 미세먼지·강풍 등 위험 요소 발생 시 위험 등급에 따른 알림 문구 출력

4.6 관리자/DB 연동 기능
- 사용자의 지역 및 설정값 저장 및 업데이트
- 백엔드 서버와 연동하여 프런트엔드 대시보드에 실시간 반영

5. 추가사항 1

라이트/다크모드 토글 버튼 추가.

6. 추가사항 2

고려사항) 지도 시각화 기능 추가(편리성, 가독성 상승).

7. 추가사항 3

고려사항) 애니메이션 추가.
